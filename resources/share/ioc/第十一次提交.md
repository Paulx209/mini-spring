【Aware接口 aware-interface 】
这章节整体的一个思路就是，通过ApplicationContextAware接口的setApplicationContext()方法 、BeanFactoryAware接口的setBeanFactory()方法将流程中的ApplicXxx 和 BeanFxxx交出去

1.新增接口 Aware 、 ApplicationContextAware 、BeanFactoryAware，后面两个主要是针对ApplicationContext 、 BeanFactory两种类来说的，前者是后者的增强。

2.这两个接口是需要类去实现的， 然后会判断类是否实现了这个接口，如果是的话，就会执行接口提供的setXXX方法，将我们的ApplicationContext、BeanFactory获取到（感应到）

3.BeanFactory 实现比较简单，直接在初始化方法的第一步就判断类是否是BeanFactoryAware的子类，如果是的话，就会执行setXxx的方法

4.ApplicationContext 是借助 BeanPostProcessor类的前置处理器这个 工具 来实现的。 实现一个before方法，然后当执行到初始化方法中的前置处理器时，就会获取到这个前置处理器，然后执行before方法，会判断这个bean是否实现了aware接口，如果是的话，就去执行setXXX方法；

ApplicationContext现在大致的生命周期如下：
【BeanDefinition】 -> 【BeanFactoryPostProcessor获取并执行】 -> 【BeanPostProcessor初始化】 -> 【单例Bean初始化过程】 -> 【实例化】 -> 【属性注入】 -> 【init初始化方法：【BeanFactoryAware接口判断】 -> 【BeanPostProcessor前置处理器】（其中会有一个是Aware的实现类，要判断是否是ApplicationContextAware的子类）-> 【afterPropertiesSet】 -> 【initMethod方法】 -> 【BeanPostProcessor after处理方法】】 -> 【bean的使用】 -> 【bean的销毁 【destroy方法】 -> 【initDestroy()方法】】
